本系统采用概念级压缩口径：

```
原始体量 = 所有已解析教材正文 total_chars 之和
整合后体量 = 所有保留/合并知识点的 definition + explanation + source_summary 字符数之和
压缩比 = 整合后体量 / 原始体量
```

当前压缩比偏高（99.8%）的原因是每本教材仅构建了前 1-3 章的知识图谱，跨教材重叠内容极少。随着 KG 覆盖章节扩大，压缩比预计将显著下降。

## 5. 重点整合案例

### 案例 1：细胞 — 跨三本教材合并

- **决策**：merge，置信度 0.95
- **来源**：组织学与胚胎学（KP1）、生理学（KP2）、生理学（KP3）
- **合并原因**：三个知识点核心定义完全一致，均指向"人体最基本的结构与功能单位"这一医学基础概念。差异仅在于补充视角不同（KP1 侧重组织结构基础，KP2 侧重生理功能执行，KP3 补充了分子层面描述）
- **合并后**：保留完整定义，source_mapping 记录三本教材来源

### 案例 2：自分泌信号 — 跨两本教材合并

- **决策**：merge，置信度 0.95
- **来源**：生理学（KP1）、病理生理学（KP2）
- **合并原因**：两者名称完全一致，核心内涵均为细胞通过分泌信号分子作用于自身进行调节。定义 1 侧重"调节机制"，定义 2 侧重"通讯方式"，属于同一概念在不同语境下的标准表述
- **合并后**：合并为统一节点，保留两种表述视角

### 案例 3：稳态（Homeostasis）— 五节点合并

- **决策**：merge，置信度 0.95
- **来源**：生理学 3 个节点 + 病理生理学 2 个节点
- **合并原因**：5 个知识点均指向生理学/病理生理学核心概念"Homeostasis"，仅在表述详略和侧重点上存在差异（如部分强调神经-体液调节机制、部分强调动态平衡特征或健康/生命活动条件）
- **合并后**：整合为单一稳态节点，覆盖调节机制、动态平衡、健康条件三个维度

### 案例 4：细胞损伤 — 跨两本教材合并

- **决策**：merge，置信度 0.95
- **来源**：病理学（KP1）、病理生理学（KP2）
- **合并原因**：两者核心病理机制高度一致，均聚焦于线粒体产能障碍引发的 ATP 耗竭及后续细胞损害。知识点 1 明确了"氧化磷酸化"具体途径与"能量危机"概念；知识点 2 补充了"过氧化物酶体"途径与自由基损伤机制
- **合并后**：保留完整损伤机制描述，涵盖氧化磷酸化障碍和自由基损伤两条通路

### 案例 5：穴位 — 跨三本教材合并

- **决策**：merge，置信度 0.95
- **来源**：组织学与胚胎学（KP1）、病理生理学（KP2）、病理学（KP3）
- **合并原因**：三个知识点均围绕同一核心概念，语义高度重合且内容互补。KP1 侧重空间结构与形成基础，KP2 强调自我更新与功能重现特性，KP3 补充了细胞互作机制与模型定位
- **合并后**：整合为完整穴位概念，覆盖结构、功能、细胞互作三个层面

## 7. 教师反馈验证

当前系统支持以下反馈类型：

1. 解释：为什么合并 A 和 B？
2. 恢复：A 不应该删除，请保留。
3. 拆分：A 和 B 不是同一个概念。
4. 合并：A 和 B 应该视为同一知识点。

### 实测案例

教师通过对话面板发送："请解释为什么合并细胞知识点。"

系统回复：merge_0001 的合并理由是三个知识点核心定义完全一致，均指向"人体最基本的结构与功能单位"这一医学基础概念。差异仅在于补充视角不同。

教师发送："请拆分稳态相关知识点。"

系统识别为 split 操作，将 merge_0004（Homeostasis 五节点合并）中的生理学视角和病理生理学视角拆分为两个独立节点，并更新 IntegrationDecision.action 为 split。

## 8. 教学完整性说明

整合后系统通过以下规则避免教学链路断裂：

1. 被其他节点 prerequisite 指向的节点默认保留。
2. 多本教材共同出现的高频核心概念默认保留。
3. 重复节点合并时保留 source_mapping。
4. RAG 仍然保留原文 chunk，可追溯到教材章节和页码。
5. 教师可以通过自然语言反馈恢复、拆分或重新合并知识点。

## 9. 已知局限

1. 当前知识图谱构建仅覆盖每本教材的前 1-3 章，完整 7 本全量抽取需要更多 API 额度。
2. 跨教材合并依赖 LLM 判断，仍可能出现误合并或漏合并。
3. 压缩比采用概念级知识库口径，不等同于生成完整新版教材。
4. 教师反馈当前支持有限 action，不是任意图谱编辑器。
5. 部署环境如不支持 FAISS，可回退到 sklearn cosine_similarity。
